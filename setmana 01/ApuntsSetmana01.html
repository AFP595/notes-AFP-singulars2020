Proves d'estil CSS

Gràcies al minijoc "CSS Diner" a https://flukeout.github.io/ he descobert que a CSS hi ha:
  • Un selector universal "A * [B,C]" que selecciona tots els elements d'una categoria dins d'un conjunt o element.
  • Un selector adjacent "A + [B,C]" que selecciona elements d'una categoria posicionats just després d'un altre element.
  • Un selector universal "A ~ [B,C]" que selecciona tots els elements d'una categoria posicionats després d'un altre element.
  • Un selector de descendència "A > [B,C]" que selecciona elements d'una categoria posicionats dins d'un conjunt o element.
  • Un pseudo-selector ordinal "[A]:nth-child(B)" que selecciona l'element que està en la posició B, comptant cada element obert des del principi de l'element seleccionat. "[A]:nth-last-child(B)" fa el mateix, però comptant des del final.
  • Un pseudo-selector ordinal "[A]:first-of-type" que selecciona el primer element d'un tipus. "[A]:last-of-type" fa el mateix, però comptant des del final.
  • Un pseudo-selector ordinal "[A]:nth-of-type(B)" que selecciona l'element d'un tipus que està en la posició B, comptant cada element obert des del principi de l'element seleccionat. "[A]:nth-last-of-type(B)" fa el mateix, però comptant des del final.
  • Un pseudo-selector "[A]:only-of-type" que selecciona l'únic element d'una categoria dins d'un conjunt o element.
  • Un pseudo-selector "[A]:empty" que selecciona elements que estan buits.
  • Un pseudo-selector "[A]:not(X)" que selecciona elements d'una categoria que no tenen un cert tipus assignat.
  • Un selector de propietat "A[atribut]" que selecciona tots els elements d'una categoria que tenen un cert atribut. A dins es poden fer evaluacions lògiques com "atribut = X", o "atribut^ = X" (el qual selecciona valors que comencen amb X), "atribut$ = X" (el qual seleccion valors que acaben amb B), i "atribut* = X" (el qual selecciona valors que contenen X en qualsevol lloc).

A més, en els pseudo-selectors ordinals amb input de posició es pot posar "even" per a seleccionar els elements parells i "odd" pels elements imparells. També es poden introduir fórmules complexes per a seleccionar elements distribuits segons regles consistents.
